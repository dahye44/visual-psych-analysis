# visual-psych-analysis

> YOLO 기반 집(House) 그림 심리 분석 모델

사용자가 그린 **집 그림(HTP, House-Tree-Person 검사의 House 항목)** 을 YOLOv8로 분석하여,
그림에 나타난 구성요소를 탐지하고 이를 심리 지표로 환산한 뒤 해석 결과를 자동으로 제공하는 프로그램입니다.

## 개요

집 그림 심리 검사(HTP)는 그림에 나타난 요소(문, 창문, 굴뚝, 태양 등)의 유무·크기·위치를 통해
그린 사람의 심리 상태를 추론하는 투사 검사입니다. 본 프로젝트는 이 해석 과정을 다음과 같이 자동화합니다.

```
집 그림 이미지
      │
      ▼
① YOLOv8 객체 탐지  →  ② 지표 계산(4개 심리 지표)  →  ③ 케이스 분류  →  ④ 해석 스크립트 출력
```

## 처리 파이프라인

### ① YOLOv8 객체 탐지
- 커스텀 학습한 `best.pt`(YOLOv8s) 모델로 집 그림의 구성요소를 탐지합니다.
- 탐지 클래스: `집전체`, `문`, `창문`, `울타리`, `길`, `태양`, `굴뚝`, `지붕`, `연기`, `나무`, `꽃`, `잔디`, `연못`
- 각 객체의 클래스명, 신뢰도(confidence), 중심 좌표(x, y), 크기(w, h)를 추출합니다.

### ② 지표 계산 (담당 파트)
탐지된 객체의 **개수·면적 비율·위치**를 바탕으로 4개의 심리 지표를 0~100점으로 환산합니다.
구현 코드: [emotion_socre.py](emotion_socre.py)

| 지표 | 평가 요소 | 한 줄 요약 |
|------|-----------|-----------|
| **Openness** (개방성) | 문, 창문, 울타리, 길 | 외부와 교류하려는 통로의 유무·크기 |
| **Vitality** (활력성) | 태양 위치, 자연물 개수, 객체 분산도 | 그림에 표출된 에너지·활동성 |
| **Warmth** (온화성) | 굴뚝·태양 크기 비율, 꽃 개수 | 따뜻함·애정 욕구의 표현 정도 |
| **Structural Stability** (현실 기반 안정성) | 지붕 비율, 집 중심 위치, 연기 | 그림의 구조적 균형과 심리적 안정감 |

> 산식의 심리학적 근거와 세부 계산 방식은 아래 [**담당 파트 상세 — 지표 계산**](#담당-파트-상세--지표-계산) 참고.

### ③ 케이스 분류
- 4개 지표 중 **50점 기준에서 가장 크게 벗어난 상위 2개 지표**를 선택합니다. (우선순위 O > V > W > S)
- 50점 이상은 대문자, 미만은 소문자로 표기하여 2글자 라벨을 만듭니다. (예: `OV`, `Ov`, `oS` …, 총 24개 케이스)

### ④ 해석 스크립트 출력
- 분류된 케이스 라벨에 매핑된 심리 해석 텍스트를 출력합니다.

## 담당 파트 상세 — 지표 계산

본 파트는 YOLO가 탐지한 객체 정보(클래스·면적·위치)를 **심리 지표 점수**로 변환하는 로직을 설계·구현하는 작업입니다.
HTP 검사에서 각 그림 요소가 갖는 심리적 의미를 점수 산식으로 모델링했습니다.

### 설계 기준

**(1) 요소별 심리적 의미** — 산식의 가·감점 방향을 정하는 근거로 삼았습니다.

| 요소 | 심리적 의미 |
|------|-------------|
| 지붕 | 내적 공상·인지 과정. 너무 크면 자폐적 공상 몰두, 너무 작으면 인지 억제 → **적정 비율일 때 안정** |
| 문 | 집과 외부 세계를 잇는 통로. 타인을 받아들이고 세상으로 나아가려는 정도 |
| 창문 | 주관적 경험·상호작용 능력에 대해 스스로 느끼는 감정 |
| 굴뚝 | 가정 내 애정 욕구·따뜻함. (과하면 좌절·우울의 내재로도 해석) |
| 연기 | 가정 내 갈등·정서 혼란 → **과도하면 감점** |
| 태양 | 따뜻함에 대한 욕구, 환경에 대한 감정 표현 |
| 크기 | 작으면 무력감·열등감, 크면 공격성·긴장 |
| 위치 | 중앙은 안정감, 한쪽으로 치우치면 위축·불안정 → **중심에 가까울수록 가점** |

**(2) 지표별 높음/낮음 해석** — 점수의 의미를 정의했습니다.

| 지표 | 높을 때 | 낮을 때 |
|------|---------|---------|
| 개방성 (Openness) | 외부 환경과 교류하고 새로운 접근을 추구 | 기존 방식을 선호 |
| 활력성 (Vitality) | 에너지를 방출하는 활동적 행동 추구 | 정적이고 잔잔한 행동 추구 |
| 온화성 (Warmth) | 이타적·공감적 | 이성적 판단 중시, 냉정 |
| 현실 기반 안정성 (Stability) | 환경 변화에 빠르게 적응 | 적응에 시간이 소요 |

### 지표별 점수 산식

각 지표는 세부 항목 점수의 합으로 계산되며, 최종적으로 **0~100 범위로 클리핑**됩니다.
기준 이미지 크기는 1280×1280이며, "면적 비율"은 *(객체 면적 / 집전체 면적)* 또는 *(객체 면적 / 전체 이미지 면적)* 을 의미합니다.

#### Openness — 개방성 (외부와의 교류 통로)

| 항목 | 점수 로직 |
|------|-----------|
| 문 | 존재 시 기본 30점. 문/집 면적비가 2% 미만이면 작을수록 최대 −20점, 4% 초과면 클수록 최대 +20점 (30~50점으로 제한) |
| 창문 | 개당 +10점(최대 5개=50점). 면적비 0.5% 이하면 최대 −2점, 1.5% 이상이면 최대 +2점 (0~30점으로 제한) |
| 울타리 | 존재 시 기본 −10점. 높이가 집의 50% 이상이면 비율에 따라 추가 최대 −10점 (외부 차단 → 감점) |
| 길 | 존재 시 +20점 (외부와의 연결 → 가점) |

→ 통로(문·창문·길)는 가점, 차단(울타리)은 감점하여 **외부와 교류하려는 경향**을 점수화.

#### Vitality — 활력성 (표출된 에너지)

| 항목 | 점수 로직 |
|------|-----------|
| 태양 위치 | 태양 중심이 이미지의 **중앙·상단**에 가까울수록 가점. (중앙도+상단도)/2 × 50 (최대 50점) |
| 자연물 개수 | 나무·꽃·잔디·연못 개수 × 5점 (최대 35점) |
| 객체 분산도 | 전체 객체 중심 좌표의 표준편차(정규화)가 클수록 가점 → 화면을 고르게 채울수록 활기 (최대 25점) |

→ 밝은 태양, 풍부한 자연물, 화면 전반에 퍼진 구성일수록 **활동적·에너지 표출**로 해석.

#### Warmth — 온화성 (따뜻함·애정 욕구)

| 항목 | 점수 로직 |
|------|-----------|
| 굴뚝 크기 비율 | 굴뚝/집 면적비 × 200 (최대 30점) |
| 태양 크기 비율 | 태양/이미지 면적비 × 400 (최대 70점) |
| 꽃 개수 | 꽃 개수 × 5점 (최대 30점) |

→ 따뜻함의 상징인 굴뚝·태양·꽃이 크고 많을수록 **애정 욕구·온화함**이 높게 표현된 것으로 점수화.

#### Structural Stability — 현실 기반 안정성 (구조적 균형)

| 항목 | 점수 로직 |
|------|-----------|
| 지붕 비율 | 지붕/집 면적비가 **40~60%일 때 최고점**, 벗어날수록 선형 감소 (50점 만점 환산) |
| 중심 위치 | 집 중심이 이미지 중앙에 가까울수록 가점, 멀어질수록 감소 (최대 30점) |
| 연기 | 연기 면적비가 작을수록 가점, 5% 초과 시 0점 (20점 만점 환산, 정서 혼란 → 감점) |
| 좌우 치우침 | 집 중심이 좌우로 치우칠수록 최대 −15점 감점 |

→ 지붕이 과하지도 작지도 않고, 집이 화면 중앙에 균형 있게 자리할수록 **현실 기반의 안정감**이 높다고 판단.

### 배치 스코어링

전체 데이터셋(JSON 어노테이션)을 순회하며 위 4개 지표를 계산하고,
결과를 [TL_score_result.csv](TL_score_result.csv)로 저장한 뒤 지표별 분포(히스토그램)와 최저 점수를 확인하여
산식이 특정 점수에 쏠리지 않도록 검증·보정했습니다.
